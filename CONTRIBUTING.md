# Contributing to A4 Aluminium Website

Thank you for your interest in contributing to the A4 Aluminium website! We welcome contributions from everyone.

## Code of Conduct

Be respectful, professional, and constructive in all interactions.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/A4-Aluminium-Website.git`
3. Create a new branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test thoroughly
6. Commit with clear messages: `git commit -m "Add: descriptive message"`
7. Push to your fork: `git push origin feature/your-feature-name`
8. Create a Pull Request

## Commit Message Guidelines

- **Add**: `Add: new feature description`
- **Fix**: `Fix: bug fix description`
- **Update**: `Update: update description`
- **Refactor**: `Refactor: code improvement description`
- **Docs**: `Docs: documentation changes`
- **Style**: `Style: formatting or styling changes`

## Code Style

### HTML
- Use semantic HTML5
- Maintain proper indentation (2 spaces)
- Include descriptive alt text for all images
- Use ARIA labels for accessibility

### CSS
- Use CSS custom properties for colors
- Mobile-first approach
- Consistent naming conventions
- Comment complex sections

### JavaScript
- Use ES6+ syntax
- Use const/let instead of var
- Add error handling
- Include comments for complex logic
- Use meaningful variable names

## Testing Checklist

Before submitting a PR:
- [ ] Code works in Chrome, Firefox, Safari, Edge
- [ ] Mobile responsive (320px - 2560px)
- [ ] All links work correctly
- [ ] Forms submit properly
- [ ] Images load correctly
- [ ] No console errors
- [ ] Accessibility tested with screen reader
- [ ] Page load performance acceptable


## Reporting Issues

Before creating an issue:
1. Check existing issues
2. Provide clear description
3. Include steps to reproduce
4. Add screenshots if relevant
5. Mention your browser/OS

## Pull Request Process

1. Update documentation if needed
2. Follow code style guidelines
3. Ensure all tests pass
4. Provide clear description of changes
5. Reference related issues
